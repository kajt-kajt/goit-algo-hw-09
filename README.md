## Висновки

Практичне тестування швидкодії двох алгоритмів:
```
Greedy algorithm performance: value 1001, 100 repeats per each value:
0.0004614729987224564
Greedy algorithm performance: value 1234, 100 repeats per each value:
0.0006822499999543652
Greedy algorithm performance: value 2345, 100 repeats per each value:
0.0010100889921886846
Dynamic algorithm performance: value 1001, 100 repeats per each value:
3.029033236001851
Dynamic algorithm performance: value 1234, 100 repeats per each value:
2.999230349989375
Dynamic algorithm performance: value 2345, 100 repeats per each value:
5.9323870740045095
```

1. Для заданого в умовах задачі набору монет обидва алгоритми видають оптимальний результат.
1. Якщо набір монет буде, наприклад, [50, 25, 10, 7, 5, 2, 1], то для суми "14" жадібний алгоритм знайде лише варіант {10: 1, 2: 2}, а алгоритм динамічного програмування - оптимальний варіант {7: 2}.
1. Складність жадібного алгоритму буде лінійною відносно кількості номіналів монет, що можна вважати константним у реальному житті.
1. Складність алгоритму динамічного програмування буде лінійною відносно суми решти (якщо кількість номіналів вважати константою).
1. На практиці у швидкодії на великих числах запропонований варіант алгоритма динамічного програмування програє на багато порядків.
